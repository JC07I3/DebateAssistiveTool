# 辯論助手 (Debate Assistive Tool)

辯論助手是一個基於 Python 與 Streamlit 開發的現代化網頁應用程式。本系統專為辯論選手與團隊設計，從最基礎的資料整理躍升為具備「多人協作」、「歷史版本控制」與「實戰講稿測算」的專業級智庫管理平台。

## 系統核心功能

### 1. 帳號權限與多人協作系統
系統全面導入嚴謹的身分驗證閘門 (採用 bcrypt 進行密碼雜湊加密)。
* **實體隔離**：每個帳號登入後所建立的盃賽與資料皆為獨立狀態，不同用戶之間的私有資料不會被互相看見。
* **協作共享**：允許系統使用者將特定的盃賽加入其他使用者的帳號，一旦加入，團隊成員便能共同管理該盃賽底下的所有論點、筆記與講稿，滿足多人團隊的備賽需求。

### 2. 強大的文本管理模組
跳脫單純的清單架構，系統將輔助文本區分為三種專業向度：
* **論據建檔 (檢視資料 / 儲存資料)**：針對零碎的證據或切入點進行記錄，包含連結、持方、標題與長篇摘要。支援透過「全文檢索」與「進階複合篩選器 (持方 / 標籤)」快速找出可用素材。
* **筆記庫**：提供全螢幕沉浸式展開的閱讀與編輯動線。支援豐富的 Markdown 語法設計，適合用來編寫賽場攻防策略或拆解對方架構的樹狀圖。
* **稿子庫**：限定為純文字輸入以專注於演說內容。除了強硬區分「正方」與「反方」外，編輯講稿時，系統會動態計算目前文稿的精準字數，協助選手在賽前準確拿捏演說分秒。

### 3. 滴水不漏的歷史版本控制 (Version Control)
在面對高強度的多人協同編輯時，難免發生內容被誤刪或覆蓋的情況。
為此，系統在背景實作了不可竄改的歷史版本追蹤。當使用者編輯一筆「論據資料」時，系統會自動快照並保存編輯前的舊版標題、持方與摘要。只要在該長文下方展開「檢視歷史編輯紀錄」，即可按時間軸倒序調閱出每一次修改的時間、修改者，以及被覆蓋掉的原始文稿。

### 4. 資料庫儀表板 (Data Dashboard)
內建互動式數據分析中心，透過 Plotly 自動將當前盃賽的複雜文字資料轉化為視覺圖表：
* 即時計算「正反方」論點平衡度的圓餅圖。
* 分析各標籤所持有戰力的柱狀圖，幫助團隊審視在哪個進攻面向上的準備仍有匱乏。

## 技術棧與架構

本系統採用標準的 MVC (Model-View-Controller) 開發模式：
* **開發語言**: Python 3.9+
* **前端框架**: [Streamlit](https://streamlit.io/)
* **資料庫管理**: SQLAlchemy (自動建立正規化資料表，具備 SQLite / MySQL / PostgreSQL 跨資料庫支援)
* **資訊安全**: bcrypt (雜湊演算法)
* **數據圖表**: Plotly & Pandas
* **表格元件**: [Streamlit-AgGrid](https://github.com/PablocFonseca/streamlit-aggrid)

## 快速啟動指南

### 本地端執行方式

1. 進入專案目錄，切換到此專案的資料夾下。
2. 安裝所有依賴套件 (推薦使用虛擬環境)：
   ```bash
   pip install -r requirements.txt
   ```
3. 啟動應用程式服務器：
   ```bash
   streamlit run app.py
   ```
   啟動後，瀏覽器將會自動開啟 `http://localhost:8501`。
   (註：初次啟動時，系統會自動在後台完成資料庫模組與資料表的建置作業，生成 `data.db` 檔案，無須額外設定。)

### Docker 容器部署

本專案已配置完整的 `Dockerfile`，支援雲端伺服器 (如 Render, AWS, GCP) 上的快速映像部署：

1. **建立 Docker 映像檔**：
   ```bash
   docker build -t debate-assistive-tool .
   ```
2. **執行 Docker 容器**：
   ```bash
   docker run -p 8080:8080 debate-assistive-tool
   ```
3. 對外開放後，前往指定端口 (預設為 8080) 即可開始使用企業級的雲端服務。
